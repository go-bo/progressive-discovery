# Scenarios

Scenarios are assembled demos that compose framework components and product screens into complete, navigable experiences for a specific lifecycle phase and product context.

## What Scenarios Are

A scenario is not a reusable component -- it's a configured instance of the framework. It selects a lifecycle phase, sets the user model state, and wires together the shell, navigation, treatments, and product screens into a coherent demo that can be walked through.

## Creating a Scenario

A scenario typically involves:

1. Setting the user model state (lifecycle phase, role, purchased products)
2. Configuring the shell with the appropriate navigation preset
3. Defining the routes and content screens for the demo flow
