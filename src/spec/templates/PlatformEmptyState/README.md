# Platform Empty State Template

Minimal empty states for platform surfaces (Reports, Workflows, Org Chart, etc.) that the user hasn't engaged with yet. Lighter-weight than full treatments — just enough to communicate what the surface does and offer a starting point.

## Pattern

Two basic variants:

1. **Template-driven** — Shows available templates with a primary CTA to create from template
2. **Blank slate** — Simple illustration, one-line description, and a "Get started" CTA

## Status

Early stage. These will be expanded as more platform surfaces are built out. The goal is to keep platform empty states simple and consistent while product treatments (unpurchased SKU, feature activation) carry the heavier discovery work.

## Components / Patterns Used

- ActionCard (empty state), FeatureCard, DiscoveryPageLayout
